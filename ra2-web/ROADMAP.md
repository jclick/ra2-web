# RA2 Web 架构重构路线图

## 概述

本文档详细说明从现有架构迁移到 ECS (Entity-Component-System) 架构的具体步骤和时间计划。

## 当前架构问题总结

### 关键问题
1. **上帝类**：GameManager 管理了所有游戏逻辑，超过 500 行代码
2. **紧耦合**：系统间直接实例化和调用，难以测试
3. **继承滥用**：Unit 基类包含过多状态和行为
4. **状态分散**：状态管理不一致，存在同步风险
5. **渲染耦合**：渲染逻辑与游戏逻辑混合

## 风险缓解

### 高风险项

| 风险 | 影响 | 缓解措施 |
|-----|------|---------|
| ECS 性能不如预期 | 高 | 早期基准测试；准备回退方案 |
| 重构周期过长 | 高 | 增量交付；每阶段可独立发布 |
| 多人同步复杂化 | 高 | 确定性模拟设计；网络组件提前设计 |
| Bug 回归 | 中 | 完整测试覆盖；A/B测试 |

### 回退策略

每个 Phase 都保持新旧系统可切换：

```typescript
// 使用配置控制
const USE_ECS = process.env.USE_ECS === 'true'

if (USE_ECS) {
  world.update(deltaTime)
} else {
  oldGameManager.update(deltaTime)
}
```

## 代码组织

### 新目录结构

```
src/
├── ecs/                        # ECS 核心框架
│   ├── core/                   # 核心类
│   ├── components/             # 组件定义
│   ├── systems/                # 系统实现
│   └── services/               # 服务层
├── game/                       # 游戏逻辑（逐步迁移到ECS）
│   └── ...existing code
├── presentation/               # 表现层
│   ├── renderer/
│   ├── ui/
│   └── input/
└── data/                       # 数据层
```

### 迁移顺序

1. **服务层**（无依赖）：PathfindingService
2. **独立系统**：EconomySystem, FogOfWarSystem
3. **核心系统**：MovementSystem, CombatSystem
4. **复杂系统**：ConstructionSystem, ProductionSystem
5. **高层系统**：TechSystem, AISystem
6. **表现层**：Renderer, UI, Input

## 成功标准

1. **功能完整**：所有原有功能正常工作
2. **性能不降级**：帧率不低于旧版本
3. **内存优化**：内存使用减少或持平
4. **可扩展性**：新增单位类型 < 30分钟
5. **可测试性**：核心逻辑单元测试覆盖 > 80%

### 术语表

- **Entity**：实体，游戏中的对象，只有ID
- **Component**：组件，纯数据，附加到实体
- **System**：系统，处理组件的逻辑
- **World**：世界，管理所有实体、组件、系统
- **EventBus**：事件总线，系统间通信机制
